# ระบบเบิกจ่ายและจัดการอุปกรณ์ (Requisition & Equipment Management System)

ระบบนี้ถูกพัฒนาขึ้นเพื่อใช้ในการจัดการแคตตาล็อกอุปกรณ์, ระบบตะกร้าสินค้าสำหรับการสั่งเบิก, และระบบการอนุมัติตามลำดับขั้น (Approval Workflow) สำหรับใช้ภายในองค์กร

## 🎯 โฟลว์การทำงาน (Workflow & User Flow)
1. **การเข้าสู่ระบบ (Authentication)**
   - ผู้ใช้เข้าสู่ระบบด้วยรหัสพนักงานและรหัสผ่าน
   - ระบบเชื่อมต่อกับ API ตรวจสอบสิทธิ์และดึงข้อมูลบุคลากร รวมถึงระดับสิทธิ์ของผู้ใช้เพื่อจำกัดสิทธิ์การเข้าถึงเมนูต่างๆ แตกต่างกันไป
2. **การเบิกอุปกรณ์ (User Requisition Flow)**
   - ผู้ใช้เข้าดูแคตตาล็อกอุปกรณ์ และเพิ่มรายการที่ต้องการลงในตะกร้า (Cart)
   - ผู้ใช้ยืนยันการขอเบิกรายการอุปกรณ์
3. **การอนุมัติ (Approval Process)**
   - **หัวหน้างาน/ผู้ดูแลระดับแรก:** ตรวจสอบและกดอนุมัติ (Approve) หรือไม่อนุมัติ (Reject) คำขอเบิก
   - **ผู้ดูแลระบบ (Admin) / ฝ่ายสต็อก:** เมื่อคำขอผ่านการอนุมัติ จะเข้าสู่กระบวนการจัดเตรียมและส่งมอบของ พร้อมปรับสถานะเอกสาร
4. **การจัดการแคตตาล็อกและอุปกรณ์ (Catalog & Equipment Management)**
   - สิทธิ์ระดับ Admin สามารถ เพิ่ม/ลบ/แก้ไข (CRUD) แคตตาล็อกและหมวดหมู่ของอุปกรณ์
   - แอดมินสามารถติดตามสถิติและออกรายงาน (Report) การเบิกจ่ายได้

## 🏗 การออกแบบระบบ (System Design)
- **Frontend / UI:** 
  - ออกแบบโดยใช้ Bootstrap เน้นให้ผู้ใช้งาน (User) สามารถเข้าใจได้ง่ายทันที
  - โครงสร้างและ Layout เน้นการทำงานแบบ Responsive ใช้ร่วมกับ Custom CSS (`style.css`)
  - ใช้ jQuery และ DataTables สำหรับการแสดงผลรายการอุปกรณ์อย่างรวดเร็วและค้นหาข้อมูลจำนวนมากได้ทันที
  - ใช้ SweetAlert2 สำหรับสร้าง Micro-interactions และ Feedback ในระหว่างทำ Action ต่างๆ เพื่อรักษา User Experience (UX) ที่ดี
- **Backend:** 
  - พัฒนาด้วย PHP ผสมผสานกับการเรียกใช้ API ยืนยันตัวตน
  - จัดการโครงสร้างแบบ Modular สำหรับ View โดยแบ่งหน้า `header.php`, `navbar.php`, `footer.php`, `condb.php` ซึ่งรวมศูนย์กลางการใช้งาน และแยกไฟล์ Business Logic (ตัวอย่างเช่นไฟล์ Action สกุล `php_*.php`) เอาไว้จัดการประมวลผลก่อน Redirect เป็นต้น
- **Database Architecture:**
  - เชื่อมต่อกับฐานข้อมูล SQL Server และ MySQL ยึดตรรกะข้อมูลพนักงาน, ข้อมูลทรัพยากรอุปกรณ์ (`tbl_equipments`, `tbl_catalogs`), และจัดการการอนุมัติใบเบิก (`tbl_orders`) เป็นหลัก

## 🛡 การรักษาความปลอดภัยและการซ่อนไฟล์ (Security Considerations)
- ระบบได้เพิ่มการจำกัดการเข้าถึงไฟล์ที่มีความเสี่ยง เช่น ไฟล์ที่มีข้อมูล Credentials ไว้ตั้งค่าฐานข้อมูล (`condb.php`) และไฟล์ Test Debug (`test.php`) โดยได้สั่งให้ Git ยกเว้นการเปิดเผยขึ้น Repository
